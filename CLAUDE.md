## Architecture

**AmgoSportInstaFeed** is an AI-powered Instagram content automation dashboard for AMGO Sports. It is a **Next.js 16 App Router** application (TypeScript) backed by **PostgreSQL via Prisma ORM**, with authentication via **NextAuth v5** (credentials + JWT). The app primarily serves as a human-in-the-loop approval interface for content generated by external **n8n workflows**.

### Content Automation Pipeline

Six n8n workflows (JSON files in `n8n-workflows/`) drive all content creation:

1. **Workflow A** (`workflow-A-content-generator.json`, 8:00 AM AEST daily + manual webhook): Generates posts using GPT-4o + DALL-E 3, uploads image to imgbb, POSTs to `/api/posts`, sends Telegram preview
2. **Workflow B** (`workflow-B-approval-publisher.json`, webhook-triggered from dashboard): Handles approve/approve_reel/reject/regenerate actions; publishes to Instagram via Blotato, generates reels via RunwayML
3. **Workflow C** (`workflow-C-shopify-sync.json`, 6:00 AM AEST Sundays): Syncs Shopify products to DB via `/api/products` — **not yet fully implemented**
4. **Workflow D** (`workflow-D-cricket-monitor.json`, 7:30 AM AEST daily): Detects Australian cricket matches via CricAPI, writes to `/api/cricket-matches`
5. **Workflow E** (`workflow-E-hashtag-intelligence.json`, 7:00 AM AEST Mondays + manual webhook): AI-generates 30 season-aware hashtags via GPT-4o-mini, atomically replaces DB via `/api/hashtags/sync`
6. **Workflow F** (`workflow-F-post-expiry.json`, 7:50 AM AEST daily): Expires PENDING posts older than 23 hours via PATCH `/api/posts/[postId]`

### Key Architectural Points

**Single ingest endpoint** — n8n POSTs new AI-generated content to `/api/posts` using `x-webhook-secret` header auth. (A previously referenced `/api/n8n-ingest` endpoint does not exist in the codebase.)

**Database access is centralized** in `src/lib/db.ts` — all Prisma queries go through functions defined there. The singleton Prisma client (with `@prisma/adapter-pg`) lives in `src/lib/prisma.ts`.

**Two auth config files exist**: `src/auth.ts` (full NextAuth config with Prisma adapter) and `src/auth.config.ts` (edge-safe config for middleware). Middleware lives in `src/middleware.ts`.

**Authentication middleware** in `src/middleware.ts` protects all routes except:
- Auth paths: `/login`, `/api/auth`
- Webhook paths (no session needed): `/api/posts`, `/api/test-post`, `/api/settings`, `/api/cricket-matches`, `/api/hashtags/sync`

**n8n triggers** are fired from the dashboard via `src/lib/n8n.ts` using Axios with `X-API-Key` header auth (n8n REST API key). Webhook paths are configured via environment variables.

**Content themes** rotate daily (7 themes defined in `src/config/constants.ts`). Workflow A **always** generates the scheduled daily theme post. If `isMatchDay=true` (set by Workflow D), it **additionally** generates a second "Match Day Inspiration" post. Both appear in the dashboard as PENDING for independent approval. Post IDs use `_DAILY` / `_MATCH` suffixes: `AMGO_YYYY-MM-DD_DAILY`, `AMGO_YYYY-MM-DD_MATCH`.

### Data Models (Prisma)

- `Post` — content calendar entries with status (`PENDING → POSTED/REJECTED/EXPIRED/FAILED`), theme, caption, hashtags, image/video URLs, DALL-E prompt, match day flag
- `Product` — Shopify products synced weekly (title, vendor, price, imageUrl, productUrl, category, inStock, featured)
- `Hashtag` — 5 categories: Brand, Product, Cricket, Regional, Trending; includes week label
- `CricketMatch` — match day tracking (date, isMatchDay, matchTitle, teams, venue, startTime)
- `User` — email/passwordHash/role (ADMIN, EDITOR, VIEWER)
- `AuditLog` — entity/action audit trail (entityType, entityId, action, performedBy, metadata)
- `Setting` — key-value app config store (dayThemes, workflow_config, etc.)

## Deployment

## n8n Workflow Details

### Workflow A — Content Generator
Runs at 8:00 AM AEST daily (22:00 UTC) and on manual webhook `/webhook/amgo-manual-trigger`.

1. Fetches `workflow_config` from `/api/settings`
2. Fetches today's cricket status from `/api/cricket-matches?date=today`
3. **Always** returns the scheduled daily theme as item #1. If `isMatchDay=true`, appends a "Match Day Inspiration" item as item #2. n8n runs the full downstream pipeline independently for each item.
4. GPT-4o generates caption (200–280 chars), 25 hashtags, DALL-E prompt, alt text (per item)
5. DALL-E 3 generates 1024×1024 image (per item)
6. Uploads image to imgbb for persistent URL (per item)
7. POSTs post data to `/api/posts` with `x-webhook-secret` header (per item) — `postId` uses `_DAILY` or `_MATCH` suffix
8. Sends Telegram photo preview with post details and post type label (per item)

**Normal day:** 1 PENDING post created (`AMGO_YYYY-MM-DD_DAILY`)
**Match day:** 2 PENDING posts created (`AMGO_YYYY-MM-DD_DAILY` + `AMGO_YYYY-MM-DD_MATCH`)

### Workflow B — Approval Handler & Publisher
Triggered by dashboard via `/webhook/amgo-approve` with payload `{ action, post_id, approved_by }`.

- **approve**: Fetches post → Blotato publishes image to Instagram → PATCHes `/api/posts/[id]` status=POSTED
- **approve_reel**: Fetches post → RunwayML generates video (60s) → Blotato publishes reel → updates DB
- **reject**: PATCHes `/api/posts/[id]` status=REJECTED
- **regenerate**: POSTs to Workflow A manual trigger webhook

All paths send Telegram confirmation alerts.

### Workflow C — Shopify Product Sync
Runs at 6:00 AM AEST Sundays (20:00 UTC Saturday). Fetches active Shopify products, normalizes data, POSTs each to `/api/products`. **Not yet fully implemented on the dashboard side.**

### Workflow D — Cricket Match Intelligence
Runs at 7:30 AM AEST daily (21:30 UTC). Calls CricAPI `/v1/currentMatches`, filters for Australian matches, POSTs result to `/api/cricket-matches` (sets `isMatchDay` true/false). Workflow A reads this 30 min later. When a match is detected, Telegram alert now reads: "An additional Match Day post will be generated alongside today's regular theme post in 30 minutes."

### Workflow E — Hashtag Intelligence
Runs at 7:00 AM AEST Mondays (21:00 UTC Sunday) and on manual webhook `/webhook/amgo-hashtag-refresh`.

1. Fetches upcoming cricket from `/api/cricket-matches?upcoming=7`
2. Builds season context (BBL, Test, off-season, etc.)
3. GPT-4o-mini generates 30 hashtags in 5 categories (Brand×5, Product×8, Cricket×8, Regional×5, Trending×4)
4. POSTs to `/api/hashtags/sync` — atomically deletes all existing and creates 30 new in one DB transaction

### Workflow F — Post Auto-Expiry
Runs at 7:50 AM AEST daily (21:50 UTC), 10 minutes before Workflow A.

1. Fetches PENDING posts older than 23 hours via `/api/posts?status=PENDING&before=<ISO>`
2. PATCHes each to status=EXPIRED
3. Sends Telegram summary of expired count
